# Palette - Spec-Kit 구현 추천 (5가지 핵심 기능)

현재 코드베이스는 Next.js 16 초기 스캐폴드 상태입니다. PRD를 기반으로 중복되지 않는 핵심 기능 5가지를 우선순위에 따라 추천합니다.

---

## 1. 인증 및 사용자 역할 시스템 (Auth & Role System)

**우선순위:** 최상

**이유:** 모든 기능의 기반이 되는 핵심 시스템. 학생/멘토/관리자 역할 분리 없이는 다른 기능 구현 불가.

**포함 기능:**
- 이메일/비밀번호 기반 회원가입/로그인
- 역할 선택 (학생/멘토)
- JWT 세션 관리
- 역할별 GNB 분기 처리
- 보호된 라우트 미들웨어

**명령어:**
```
/speckit.specify 이메일/비밀번호 기반 회원가입 및 로그인 시스템 구현. 사용자는 가입 시 학생 또는 멘토 역할을 선택해야 한다. 로그인 후 역할에 따라 다른 GNB를 보여주며, 인증되지 않은 사용자는 갤러리 썸네일만 볼 수 있고 상세 페이지 접근 시 로그인 페이지로 리다이렉트된다. Phase 1에서는 Mock 데이터와 Zustand를 사용하여 인증 상태를 관리한다.
```

---

## 2. 갤러리 커뮤니티 (Gallery Community)

**우선순위:** 상

**이유:** 서비스의 핵심 콘텐츠. 학생들이 작품을 공유하고 피드백을 받는 메인 공간.

**포함 기능:**
- 작품 그리드 리스트 (무한 스크롤)
- 작품 업로드 (이미지 최적화)
- 작품 상세 보기
- 댓글 시스템 (멘토 댓글 강조)
- 좋아요 기능
- 스크랩(북마크) 기능

**명령어:**
```
/speckit.specify 미대 입시생을 위한 작품 갤러리 커뮤니티 구현. 학생은 자신의 그림을 업로드할 수 있고, 모든 사용자는 작품에 댓글과 좋아요를 남길 수 있다. 멘토의 댓글은 배경색과 멘토 배지로 시각적으로 구분된다. 학생은 마음에 드는 작품을 스크랩하여 마이페이지에서 확인할 수 있다. 갤러리는 무한 스크롤로 구현하며, 이미지는 썸네일 최적화를 적용한다. Phase 1에서는 Mock 데이터를 사용한다.
```

## 3. 1:1 코칭 시스템 (Coaching System)

**우선순위:** 상

**이유:** 서비스의 핵심 수익 모델. 크레딧 기반 유료 피드백 시스템.

**포함 기능:**
- 멘토에게 1:1 질문하기 (크레딧 차감)
- 24시간 SLA 타이머 (미답변 시 자동 환불)
- Canvas 기반 피드백 도구 (브러시/도형/텍스트)
- 원본/피드백 이미지 비교 뷰어
- 추가 질문 (1회)
- 48시간 후 세션 자동 종료

**명령어:**
```
/speckit.specify 크레딧 기반 1:1 코칭 시스템 구현. 학생이 멘토를 선택하고 그림과 질문을 업로드하면 크레딧이 차감되며 24시간 카운트다운이 시작된다. 멘토는 Canvas 도구로 학생의 그림 위에 직접 피드백을 그려서 답변한다. 24시간 내 미답변 시 크레딧 자동 환불. 학생은 답변 후 1회 추가 질문 가능. 48시간 동안 추가 질문이 없으면 세션 자동 종료. 원본과 피드백 이미지를 토글 또는 오버레이로 비교할 수 있다.
```

## 4. 멘토 시스템 (Mentor System)

**우선순위:** 중

**이유:** 플랫폼 품질 보장을 위한 멘토 검증 체계.

**포함 기능:**
- 멘토 프로필 페이지 (대학, 스타일 태그, 예시 작품)
- 멘토 디렉토리 (필터/검색)
- 멘토 심사 프로세스 (서류 제출)
- 관리자 승인/반려 워크플로우
- 승인 전 기능 제한 처리

**명령어:**
```
/speckit.specify 멘토 검증 및 디렉토리 시스템 구현. 멘토 지망자는 증빙 서류를 업로드하고 승인을 기다린다. 승인 전에는 프로필 설정만 가능하고 다른 메뉴 접근 시 승인 대기 중 안내가 표시된다. 관리자는 서류를 검토하여 승인 또는 반려(사유 필수)할 수 있다. 승인된 멘토는 배지가 활성화되고 멘토 디렉토리에 노출된다. 학생은 대학, 스타일 태그로 멘토를 필터링하여 검색할 수 있다.
```

## 5. 실시간 모의고사 (Live Exam)

**우선순위:** 중

**이유:** 차별화된 핵심 기능. 실제 입시 환경과 유사한 실시간 경험 제공.

**포함 기능:**
- 모의고사 생성 (멘토)
- 시험 스케줄 및 참가 신청
- 서버 시간 기반 실시간 동기화
- 시작 시 주제 이미지 자동 공개
- 종료 시 업로드 버튼 비활성화
- 지각/중도 입장 허용
- 제출작 필수 피드백 (멘토)

**명령어:**
```
/speckit.specify 실시간 모의고사 시스템 구현. 멘토가 시작 시간, 종료 시간, 주제 이미지를 설정하여 모의고사를 생성한다. 학생은 모의고사에 참가 신청하고 시작 시간이 되면 새로고침 없이 주제 이미지가 자동 공개된다. 종료 시간이 되면 업로드 버튼이 비활성화된다. 시험 시작 후에도 종료 전까지 입장 및 제출이 가능하다. 주최 멘토는 모든 제출작에 피드백을 달아야 채점 완료 상태가 된다. 서버 시간 기준으로 동기화하며 WebSocket 또는 Polling을 사용한다.
```

---

## 구현 순서 추천

1. **인증 및 사용자 역할 시스템** - 모든 기능의 기반
2. **갤러리 커뮤니티** - 핵심 콘텐츠 및 트래픽 유입
3. **멘토 시스템** - 코칭 시스템 전제 조건
4. **1:1 코칭 시스템** - 핵심 수익 모델
5. **실시간 모의고사** - 차별화 기능

---

## 참고 사항

- 모든 Spec은 Phase 1 (Mock Data) 기준으로 작성됨
- 각 Spec 완료 후 `/speckit.plan` -> `/speckit.tasks` -> `/speckit.implement` 순서로 진행
- Backend 통합은 Phase 2에서 Supabase로 전환 예정
